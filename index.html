Statsball

Introduction

Statsball is a web application project (still ongoing), I designed and built a comprehensive system to collect, process, and visualize football data from various sources. The goal was to create a powerful platform for match analysis and player performance prediction.

I started by developing a robust ETL pipeline using Django and Python to efficiently integrate data from StatsBomb API, Opta, and other providers. The pipeline handles over 1500 events per match with near-perfect accuracy. To ensure data consistency, I engineered a flexible data model that standardizes diverse event types across sources, allowing for seamless integration of new data feeds.

One of the key features I implemented is an interactive data visualization module using React and D3.js. This enables users to explore match data through intuitive heatmaps, pass networks, and player movement charts. The visualizations provide valuable insights for real-time match analysis.

Carry Algorithm

Introduction and Purpose

According to Opta, a carry is defined as "a player moving the ball five metres or more." However, the data I was working with either missed some carries or lacked a carry type altogether. This led me to develop my own method for identifying carries using event data.

Comparison

Images showing carries by Marcus Rashford vs. Brentford (Home) in the 2024/25 Premier League season.

Only carries exceeding 3m were considered for comparison purposes.

Using raw event data, I achieved a 100% match rate against both Opta (which uses a 5m limit) and The Athletic. Notably, both The Athletic and Opta missed a carry by Rashford at 88:50, which I verified with video evidence, as shown below. This carry was crucial, culminating in a shot.
